    • Starting the Code

    The first step to making anything in html is to start with the DOCTYPE. The DOCTYPE is special meaning it's different from the rest of the code. Before you start working on your code, you need to specify what language you are using so the browser can understand what the code means. The DOCTYPE is not a tag, it's more of a dictionary for the browser to understand the language of the code and properly transfer it onto the page. In order to insert the DOCTYPE all you need to do is type "!" into the first line and hit tab in html. This automatically gives you a template to work with as well as specifying the doc type.

    Inserting Headers

    Next is to actually work on the pages for your site. The first thing you need to do is give a title to the page, and in order to do that you have to insert a header. The way you do this is with the header tag. This tag looks like this: "< h1 >< /h1>" Between the starting tag and ending tag you can write whatever you want the title to be. There are also different different headers, you can go between them by changing the number next to h (ie. "< h2 >").

    Making Lists

    Lists are something I didn't use very often but nevertheless, are still important and part of the basics of coding. Now first thing you need to know about lists is that there are two different types of lists: the unordered list (labled as "ul") and ordered list (labeled as "ol"). The unordered list will have bullet points for the list items, and the ordered list will have numbers instead. The list needs an openning tag and a closing tag. The openning tag looks like this "< ol >" for the ordered list and "< ul >" for the unordered list. The closing tag looks identical to those, but just with a backslash in front of the letters: "< /ol >" or "< /ul >". However just writing these tags and putting words in them will not make them a list. In order to tell the code that you want to make it a list then you have to specify that the words you are typing are "list items". In order to do this you make an openning and closing tag for list items: "< li >" and "< /li >". Here is a real-life example of this:

    Styling with CSS

    For me, css was very complicated, not only because it is but also because it's a whole other language. I can't explain everything I did here, but I can try to explain the basics and concepts of css.

    The first thing you need to understand about css is that it's only used for styling, meaning that you still need to know html in order to use css. Also in order for html to recognize taht you are using css, you need to tell it that you are using a different language. There are 3 ways to style with css, there is external, internal, and inline.

    Each changes in specificity, the most specific is inline. This is where you go into a tag and put a style attribute into the tag in order to style that one piece of code (the style="" is refered to as an attribute). Inline styling especially good when you only want to style one tag and don't need to go through the work of making a seperate css page, or a style tag.

    The other way to do this is the internal styling where you would write a style tag into the head of the html code and then type css, nesting in the style tags (nesting means to insert something in between an openning and closing tag). The style tag tells the html code that you are going to css in there.

    The difference between inline and the internal is that is in internal is that you need to specify what tag you are trying to style. As you can see in the image you need to specify the tag you are trying to style, you do this by just typing what the tag is (without the less "<" or ">"). Next, you need to add a curly open and closing bracket, between these brackets you need to write what you are trying to change (in this case the color) and what you want to change it into (in this case its blue). This type of styling is good when you are trying to style a lot on a single page and don't want to go in one-by-one trying to style every tag.

    The last and most complicated but the most useful is the external styling. Here you create a seperate css page, then link that page into the head of the html page you want to style using a link tag. For my about me page I made a seperate css page and called it "aboutme.css" and inserted links into each of the 4 pages. This way I don't have to retype the same styling for every page, instead I can just insert the link into each page (through copy & paste) and all of the pages will have the same styling. This is useful because it saves a lot of time and can help when you are trying to style multiple pages at once.

    In the link, all you need to do is write the name of the page into the href. Once you have done this and inserted the link into all the pages you want, then you can go into css page and style whatever you want and it'll transfer onto the pages with the links.
